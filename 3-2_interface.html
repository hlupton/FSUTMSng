Model Calibration and Validation

In February 2022, the Model Task Force adopted the validation guidance prepared for FDOT as part of the process of developing standards for FSUTMS. This document provides some needed specificity on guidelines for model validation and includes the following elements:

  • The preferred model validation process, including considerations and best practices, types of validation checks, the use of targets and guidelines, the development of a model validation plan, and validation of model input data
  • The process of validating the demand components of trip based and activity based model
  • The validation process for highway and transit assignment
  • The process for temporal validation and sensitivity testing

The validation guidance document does not distinguish among guidance for different stakeholders, such as model developers, appliers, and reviewers. However, the NextGen technical team sees the value in providing specific guidance for each of these groups even though there is significant overlap in the guidance. Therefore, the summary of the validation guidelines is presented below for each group.

It is important to recognize that while the list of checks should be performed in all applicable cases, numeric targets should be treated as guidelines, not standards. The points below can be treated as a checklist of tasks to perform, but numeric guidelines are not pass/fail tests of model validity. See the validation guidance document for more information.


Guidance for Model Developers

General
  • Develop a model validation plan prior to model development
  • Validate/check model input data
  • Validate all components of the model
  • Perform temporal and sensitivity testing

Comparisons of base year model results to observations
Trip Generation
  • Check number of trips per household for reasonableness
    • 9 to 12 if non-motorized trips are included
    • 8 to 11 if non-motorized trips are excluded
  • Percentages of trips by purpose
  • HBW attractions per employee: 1.2 to 1.5
Trip Distribution
  • Compare modeled average trip lengths by trip purpose to observed.
    • Target: Modeled trip lengths by trip purpose within five percent of observed
  • Compare trip length frequency distributions by trip purpose to observed.
    • Target: Coincidence ratios by trip purpose of at least 70 percent
  • Compare modeled and observed trip tables by purpose at a district level
  • Compare modeled and observed intrazonal trips by purpose.
    • Target: Modeled percentages of intrazonal trips with three percentage points of observed
Mode Choice
  • Create targets for observed trips by mode and purpose and compare model results for reasonable match
Time of Day
  • Compare modeled percentages of trips by time period for each trip purpose to observed percentages
Activity-based model components
  • Include all checks for trip-based components above, using tours rather than trips as appropriate.
  • Additional checks (compare to observed data, segmented by person/household characteristics)
    • Percentages of workers who go to work and students who go to school
    • Percentages of people who do not travel
    • Simulated workplace locations (compare to employment)
    • Number of tours per person
    • Tour/activity durations
    • Stops per tour by activity type
Highway Assignment
  • Compare modeled VMT on links with counts to observed VMT, summarized by functional class.
    • Targets: Per Table 4 of guidance document


    Functional Class Percentage VMT Difference
    Freeways/Expressways 7%
    Principal Arterials 10%
    Minor Arterials 15%
    Collectors 20%
    All Links 2%


  • Compare volumes on screenlines and cutlines to observed.
    • Targets: Per Table 5 of guidance document

    Volume Range Percentage Volume Difference
    1 - 20,000 45%
    20,000 - 50,000 30%
    50,000 - 100,000 25%
    100,000 - 200,000 20%
    200,000+ 15%


  • Compute percentage RMSE between modeled and observed link volumes, summarizing by volume group.
    • Targets: Per Table 6 of guidance document

    Volume Range Percentage Volume Difference
    0 - 5,000 100%
    5,000 - 10,000 45%
    10,000 - 15,000 35%
    15,000 - 20,000 30%
    20,000 - 50,000 25%
    50,000+ 20%


Transit Assignment
  • Compare modeled transit ridership to observed ridership, summarized by groups of routes.
  • Compare modeled transfer rates to observed.
  • Compare modeled volumes to observed for major transit rider subgroups
  • Compare modeled volumes on “major” screenlines to observed.

Temporal validation and sensitivity testing
  • If data are available, run model for a year prior to the base year, a year after the base year but before the present, or a year in the future and compare the results to the base year results for reasonableness.
  • Perform sensitivity tests consistent with the types of application for which the model is expected to be used.
    • Changes in costs (auto operating/fares/tolls)
    • Changes in transit level of service
    • Changes in roadway capacity
    • Changes in auto ownership
    • Aging of the population
    • Brownfield development
    • Reallocation of population or employment within the region
    • Increased indidence of working at home

Guidance for Model Appliers

  • Review model validation plan and validation documentation prior to model application.
  • Check model validation results (the checks listed above for model developers) with a focus on model components relevant to the application for which the model is being used
    • Consider performing checks that may not have been done in the original validation.
  • If appropriate, perform additional sensitivity testing specific to the application for which the model is being used.

Guidance for Model Reviewers

  • Review model validation plan and validation documentation prior to model application.
  • Check model validation results (the checks listed above for model developers).
    • Consider performing checks that may not have been done in the original validation
    • Identify areas where the validation results may require further attention.
    • Consider performing additional temporal/sensitivity tests as appropriate
